# Cover Letter Draft: Computers & Security

Submission status: pending human portal submission

Dear Editors of *Computers & Security*,

We are submitting the manuscript **"Antigence Subnet v10.0: Non-Determinism
Hardening, Weighted-Ensemble Policy, and a Simulation-Bounded Release Path for
Decentralized AI Verification"** for consideration as an original research
article.

This manuscript presents a security-focused result from a decentralized AI
verification system built on Bittensor. The core contribution is not a claim
that more agentic complexity automatically improves security. Instead, the work
shows the opposite under measurement: a thin swarm-style expansion was tested,
failed the predeclared gate, and was rejected. The shipped v10 path therefore
focuses on the stronger security result: explicit non-determinism hardening,
weighted-ensemble calibration, a clear operator policy layer
(`operator_multiband`), and an honest simulation-bounded deployment contract.

We believe the paper fits *Computers & Security* for three reasons:

1. It treats AI output verification as a security problem rather than only a
   machine-learning benchmark problem.
2. It contributes an evidence-bound negative result on architectural complexity,
   which is valuable for practitioners facing pressure to add consensus layers
   without proof of benefit.
3. It emphasizes operational honesty by separating observed simulation-backed
   deployment evidence from intentionally deferred real-network claims.

The submission package in this repository includes the manuscript source, a
supplementary archive grounded in the Phase 92-95 evidence chain, and a
traceability report for the final manuscript. No part of this package should be
interpreted as a completed portal submission until the human upload step is
performed.
